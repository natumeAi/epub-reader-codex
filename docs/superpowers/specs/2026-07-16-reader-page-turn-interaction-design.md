# 阅读器混合跟手翻页设计

日期：2026-07-16

状态：设计已由用户逐节确认，尚未编写实施计划

## 摘要

将当前触摸设备上的“纯色遮罩擦过”翻页改为混合交互：左右点按和桌面方向键触发真实正文自动滑页，手机和平板横向拖动时正文与手指一比一跟随。拖动距离或速度任一达标即完成一页翻页，否则回弹到原页；两页交界处使用轻量柔和阴影，不加入 3D 卷页。

实现以 epub.js 0.3.93 的 `continuous` 分页管理器为基础，启用其 Snap 能力来维持相邻视图和对齐语义。新增 React 手势控制器和无 React 的 epub.js 适配层，后者是唯一允许读取 manager、scroller 和 snapper 内部接口的模块。最终阅读位置仍以落页后的 `relocated` 为准；内部能力不可用时自动降级为现有单次 `next/prev`。

## 背景

当前阅读器使用 epub.js 默认 manager，配置为 `flow: 'paginated'`、`spread: 'none'`、零列间距。`ReaderView` 维护两阶段翻页状态：先让当前页离场，再调用一次 `rendition.next()` 或 `prev()`，等待 `relocated` 后让新页入场。

桌面端会移动整个 EPUB 容器；触摸端为了避免 iPhone 和安卓平板移动全屏 iframe 时卡顿，已改为只移动主题色遮罩。该优化降低了设备负载，却造成当前主要问题：用户看到的是空白颜色擦过，而不是书页正文移动。

本设计有意替换触摸端遮罩方案，同时保留它已经建立的正确性约束：不能重复翻页、不能在动画中写入错误进度、失败时必须仍能阅读。

## 已确认的产品决策

- 采用混合交互：点按自动滑页，触摸拖动全程跟手。
- 跟手拖动只支持手机和平板触摸输入；鼠标拖拽和触控板手势不在本轮范围内。
- 桌面保留左右点击与方向键，并使用自动滑页。
- 拖动完成采用“距离或速度任一达标”；两者都未达标时回弹。
- 页缘使用窄幅柔和阴影，不使用 3D 卷曲、高光纸张或透明度淡入淡出。
- 一次点按或手势最多翻一页。
- 不提供用户可见的翻页方式或动画开关。
- 目标实机为 iPhone 14 Pro Max，以及骁龙 8 Gen 3、8GB RAM 的联想小新 Pro GT 平板。

## 目标

- 触摸拖动时始终显示真实 EPUB 正文，正文位置与手指位移连续对应。
- 点按、键盘和拖动落页使用一致的页面方向、页缘效果和最终位置语义。
- 保持 CFI、页码、章节、阅读百分比和继续阅读位置准确。
- 在目标设备上连续翻页无白屏、明显闪烁或持续掉帧。
- 将 epub.js 私有接口限制在一个可检测、可降级、可单测的边界内。
- 保持主题、字体、行距、字距、页边距、目录跳转和 PWA 恢复行为兼容。

## 非目标

- 鼠标跟手拖拽、触控板横向手势或多指手势。
- 3D 卷页、纸张弯折、页面截图或 DOM 快照。
- 同时维护两套 Book 或 Rendition。
- 一次手势连续跨越多页，或快速滑动后的惯性多页翻动。
- 新增翻页设置项或服务端配置。
- 为未知低端设备承诺相同帧率；能力或运行时对齐失败时会降级，但不根据设备型号预判性能档位。
- 修改阅读进度 API、数据库或飞牛 NAS 部署方式。

## 方案选择

### 采用：Continuous manager + Snap 适配层

epub.js 0.3.93 已提供 `manager: 'continuous'`、`flow: 'paginated'` 和 `snap` 选项，并在自带示例中组合使用。Continuous manager 会在视口附近维护相邻 section view，Snap 会在未对齐时抑制最终位置上报。这使真实相邻正文可以随 scroller 移动，并让落页后的 `relocated` 继续成为位置来源。

应用仍需要一个薄适配层，原因是现有全屏手势层负责左、中、右点击以及控制层显隐，不能直接把所有触摸交给 iframe。适配层统一管理 continuous manager 的 scroller、分页宽度、方向归一化和对齐动画，并隐藏 epub.js 私有对象。

### 未采用：双 Rendition 轮播

同时渲染当前页和相邻页能够绕开 manager 的滚动内部结构，但需要同步两套排版设置、CFI 和生命周期，内存与 iframe 数量也显著增加。章节边界和上翻预备页尤其复杂，不符合本轮的可靠性目标。

### 未采用：页面截图或 DOM 快照

快照可以让底层提前导航，但生成过程可能成为新的卡顿来源。复杂字体、图片、主题和 iframe 内容也容易产生失真；未达阈值回弹时还需要恢复底层位置。该方案不能稳定满足“真实正文始终可见”。

## 架构

### `client/src/components/reader/ReaderView.jsx`

`ReaderView` 继续作为阅读器组合层，并保留全屏手势层。它只负责把以下输入交给控制器：

- 左右区域点按；
- 中间区域控制层显隐；
- `pointerdown`、`pointermove`、`pointerup` 和 `pointercancel`；
- 桌面左右方向键。

`ReaderView` 不再读取 epub.js 的 `scrollLeft`、manager 或 snapper，也不再维护当前 `out/in` 空白遮罩时序。它根据控制器暴露的阶段、方向和进度渲染一个柔和页缘元素。

### `client/src/hooks/usePageTurnController.js`

新增 hook 作为交互状态机，状态限定为：

- `idle`：可接受新输入；
- `pending`：已按下但尚未锁定横向或纵向；
- `dragging`：触摸横向拖动已锁定；
- `settling`：正在完成翻页或回弹；
- `basic`：增强能力不可用，本次会话使用基础翻页。

控制器负责方向锁定、速度采样、距离判定、边界阻尼、一次一页限制和并发保护。它不理解 CFI、章节、主题或保存 API，也不直接写入 epub.js DOM。

控制器仅在手势开始、阶段变化和结束时更新 React 状态。逐帧拖动通过适配层直接写 scroller，并通过 CSS 自定义属性更新页缘进度，避免每帧触发 React 重渲染。

### `client/src/utils/epubPageTurnAdapter.js`

新增无 React 依赖的适配层，是唯一允许接触下列对象的模块：

- `rendition.manager`；
- manager 的滚动容器；
- manager 的 layout、direction 和 RTL scroll type；
- Snap 的对齐状态或能力。

适配层提供清晰的高层操作：检查能力、读取页宽和稳定位置、开始拖动、按位移移动、动画到指定相邻页、动画回原页、边界阻尼、取消和销毁。

所有目标位置都从手势开始时的已对齐原点计算，而不是从不断变化的当前位置累加。这样即使用户拖过半页，也不会因为 Snap 四舍五入再额外前进一页。

Snap 负责 continuous manager 的相邻视图预备和“未对齐时不报告最终位置”语义。为了满足动态 120–220ms 时长、ease-out 收尾和一次最多一页，适配层使用自己的 rAF 对齐动画，目标只能是稳定原点或稳定原点正负一个页宽；不直接调用可能基于当前位置再次取整的多页 Snap 操作。

现有全屏手势层始终位于 EPUB 容器之上，是唯一的应用输入面。应用不再给 iframe 注册另一套拖动控制器，避免同一次触摸同时驱动外层控制器和 Snap 自带的触摸监听。Snap 的 scroller 监听仍用于跟踪对齐状态和抑制中间位置上报。

适配层必须处理 LTR、RTL 和浏览器 RTL scroll type 的差异；若无法得到有效页宽、方向或对齐原点，能力检查失败并进入基础翻页。

### `client/src/hooks/useEpubRendition.js`

Rendition 改为使用 continuous manager、paginated flow、单页 spread 和 Snap。现有 content hook 继续把主题与排版设置应用到每个新建的相邻 view；`rendered` 后的兼容性重套逻辑保留。

`useEpubRendition` 仍负责书籍载入、初始 CFI、目录、`relocated`、进度计算和生命周期恢复。它向 `ReaderView` 暴露 rendition 与增强翻页是否就绪，不接管具体手势。

### `client/src/styles/reader.css`

移除触摸端全屏主题色 sheet 的翻页职责。新增单个页缘层，使用 transform 和窄幅阴影表达前后页关系。页缘位置由方向和 `--reader-page-turn-progress` 驱动；不复制正文、不使用 backdrop-filter、不使用大范围模糊或 3D 变换。

`will-change` 仅在 `dragging` 和 `settling` 阶段启用，结束后立即移除。

### 既有进度链路

`usePageProgress`、页码展示和阅读进度 API 的职责不变。Continuous manager 在拖动未对齐期间不应产出最终位置；对齐后的 `relocated` 更新：

- `currentCfiRef`；
- 当前章节 href；
- 分页页码；
- 阅读百分比；
- 两秒防抖的阅读进度保存。

`relocated` 处理器在写入上述状态前必须确认适配层处于稳定对齐位置。`pending`、`dragging` 或未对齐的事件不进入进度链。回弹可能重新报告与原页相同的 CFI，但不得产生新的阅读位置或百分比跳变。

## 导航来源约束

该设计明确区分两条导航路径：

- **增强路径**：Continuous manager 的 scroller 对齐到相邻页就是一次真实翻页。落页后不得再调用 `rendition.next/prev`，否则会重复前进。
- **基础降级路径**：不直接移动内部 scroller，只调用一次 `rendition.next/prev`。

这条规则取代项目当前“所有真实翻页只能由 `next/prev` 执行”的约定，但只限于本设计的 continuous manager 增强路径。两条路径在同一次操作中不能混用。

## 交互与数据流

### 点按与键盘

1. 控制器确认当前处于 `idle`，并取得稳定原点、页宽与方向。
2. 中间点按只显隐控制层，不启动翻页。
3. 左右点按或方向键先检查书籍边界和适配层能力。
4. 增强路径把 scroller 从原点动画到相邻的一个页宽；页缘同步移动。
5. 落页对齐后等待 `relocated`，更新现有进度链，并回到 `idle`。
6. 能力不可用时改为基础路径，调用一次 `next/prev`。

### 触摸拖动

1. 仅 `pointerType === 'touch'` 能进入跟手路径；多指触摸不启动翻页。
2. `pointerdown` 记录起点、时间、稳定原点和指针标识，状态进入 `pending`。
3. 位移达到 10px 后判断方向。横向绝对位移必须至少为纵向的 1.2 倍；纵向胜出时放弃翻页并交还浏览器。
4. 横向锁定后进入 `dragging`。适配层在每个 animation frame 最多写入一次 scroller，正文与有效横向位移一比一移动。
5. `pointerup` 计算总距离和最近一段速度。距离或速度任一达标就落到相邻页，否则回到原点。
6. `settling` 期间忽略新输入；动画结束并稳定对齐后才回到 `idle`。

### 初始手感参数

- 方向锁定距离：10px。
- 横向优势比例：`abs(dx) >= abs(dy) * 1.2`。
- 距离阈值：视口页宽的 28%，并限制在 72–160px。
- 速度阈值：0.45px/ms；快速轻扫可以在距离不足时完成翻页。
- 点按自动滑页：目标约 180ms。
- 拖动后的落页或回弹：按剩余距离计算，并限制在 120–220ms。
- 收尾缓动：快速收敛的 ease-out；拖动阶段不使用缓动。
- 首尾页阻尼：最多移动 28px，随后回弹。
- 每次操作最多跨一个页面宽度。

这些数值集中定义为可测试常量；实机验证可以在不改变交互规则的前提下微调。

## 柔和页缘

页缘只在 `dragging` 或 `settling` 时显示。它位于正在进入页面的交界处，方向与下一页或上一页保持一致，并随归一化拖动进度从视口边缘移动。

视觉限制如下：

- 只使用窄渐变和小范围阴影；
- 不遮盖整页，不改变正文 opacity；
- 不使用 3D perspective、滤镜模糊或页面弯折；
- 主题颜色来自现有阅读器 CSS 变量，夜间主题降低阴影对比度；
- 回弹和完成后立即隐藏，不残留合成层。

## 错误处理与降级

### 初始化能力检查

增强模式至少要求以下条件成立：

- rendition 和 continuous manager 已创建；
- manager 滚动容器存在；
- layout 能提供有限且大于零的分页宽度；
- 当前滚动位置可以归一化为稳定页原点；
- 当前书写方向能被适配层处理。

任一条件失败时，本次阅读会话进入 `basic`。该降级不向用户显示错误，因为基础翻页仍然可用。

### 相邻页未就绪

Continuous manager 应提前维护相邻 view。若锁定方向时对应内容仍未就绪，当前页只允许有限阻尼，不露出背景空白；松手后先对齐回稳定原点，再走一次基础翻页。下一次操作可重新进行能力检查。

### 超时或失去对齐

落页后最多等待 1200ms 的 `relocated`。超时、无效位置或 scroller 不能对齐时：

1. 停止动画并清理页缘；
2. 使用最近一次稳定 CFI 恢复 rendition；
3. 将本次会话切换到 `basic`；
4. 继续允许用户翻页和关闭阅读器。

### 生命周期中断

`pointercancel`、视口旋转、设置重排、页面切到后台和组件卸载都必须取消当前手势。控制器先恢复最近稳定页面，再允许 resize、`display(currentCfi)` 或销毁操作继续。所有 rAF、计时器、pointer capture 和事件监听必须在取消或卸载时释放。

### 减少动态效果

系统启用 `prefers-reduced-motion: reduce` 时，不启用跟手移动和页缘动画。点按、触摸滑动识别和方向键最终都只触发一次无动画基础翻页，数据语义不变。

## 性能设计

- 拖动热路径不调用 React `setState`；每帧最多写一次 scroller 和一个 CSS 自定义属性。
- 不创建第二个 Rendition、Canvas 或页面快照。
- 依赖 continuous manager 的 view trim，不在应用层永久保存相邻 iframe。
- 不在 pointermove 中读取正文布局；页宽、原点和边界在手势开始时缓存。
- DOM 读取与写入分阶段，避免交替触发布局。
- 进度保存仍为两秒防抖异步请求，不阻塞动画。
- 飞牛 NAS 只影响 EPUB 首次下载；书籍已完整读取为 ArrayBuffer 后，翻页排版和动画都在访问设备上执行。

## 测试设计

当前仓库有 Playwright 移动端验证脚本，但尚无可执行的前端单元测试命令。实施前必须先完成仓库已有的测试基础计划，或在本功能计划中建立最小的 Vitest、jsdom 和 React Testing Library 环境。

### 纯逻辑单元测试

- 10px 方向锁定及横纵比例判断。
- 距离阈值在手机和宽屏平板上的 72–160px 限制。
- 正向和反向速度计算。
- 距离达标、速度达标、两者均未达标三种结果。
- 首尾页阻尼上限。
- 一次手势只能得到 `-1`、`0` 或 `1` 页结果。

### 适配层单元测试

- continuous manager 能力检测成功与各失败分支。
- LTR、RTL default、RTL negative 三种滚动位置换算。
- 拖动位置始终由稳定原点计算，不累积漂移。
- 完成翻页精确对齐一个页宽，回弹精确回到原点。
- 外层手势控制与 Snap 监听不会对同一次触摸产生双重位移。
- 超时恢复稳定 CFI 并切换基础模式。
- 取消和销毁清理 rAF、计时器及监听器。

### Hook 与组件测试

- 左、中、右点按分别执行上一页、显隐控制层和下一页。
- 触摸拖动跟手，鼠标拖拽不进入跟手模式。
- 距离完成、速度完成和未达标回弹。
- `settling` 期间快速重复输入不能造成双重翻页。
- 键盘左右键仍工作，编辑控件和设置面板打开时不翻页。
- reduced motion 只调用一次基础导航且无视觉等待。
- 适配层失败时阅读器保持可用。

### Playwright 集成测试

- 在手机触摸视口中拖动到中间位置，验证真实 EPUB scroller 连续改变且没有全屏纯色 sheet。
- 达标松手后 CFI 只前进或后退一页；未达标回弹后 CFI 不变。
- 快速轻扫在距离不足时仍完成一页。
- 跨章节、首尾页和快速连续操作不出现空白、重复导航或锁死。
- 白色、暖色、护眼和夜间主题下页缘与正文一致。
- 修改字号、字体、页边距、行距和字距后，相邻 view 样式一致且当前位置稳定。
- 目录跳转、页码、保存、重新打开继续阅读和 PWA 前后台恢复继续通过。
- reduced motion 模式不执行跟手动画。

### 实机验证

在 iPhone 14 Pro Max 和联想小新 Pro GT 上分别验证：

- 点按与拖动两种方式连续翻动至少 50 页；
- 普通文本、图片较多页面和章节边界均无白屏或明显闪烁；
- 正文在拖动中没有可感知的落后或跳动；
- 回弹、快扫和首尾页阻尼符合规则；
- 长时间翻页后相邻 iframe 数量不持续增长；
- 切后台再恢复、旋转和调整排版后仍能继续翻页。

Playwright 的移动端模拟只能验证交互与数据正确性，不能替代这两台设备上的 GPU 合成流畅度验收。

## 验收标准

- 触摸拖动全过程显示真实正文，不出现当前纯色遮罩擦屏。
- 点按、键盘和达标拖动每次只移动一页。
- 未达标拖动、首尾页拖动和取消事件最终返回稳定页面。
- 最终 CFI、页码、章节与阅读百分比正确，重新打开能回到最后稳定页。
- 主题和排版设置在当前及相邻 view 中一致。
- 目标实机连续翻动 50 页无白屏、明显闪烁、输入锁死或持续 iframe 增长。
- epub.js 内部接口变化或运行异常时自动降级，基础翻页仍可用。
- reduced motion 模式无跟手或页缘动画，但翻页结果一致。
