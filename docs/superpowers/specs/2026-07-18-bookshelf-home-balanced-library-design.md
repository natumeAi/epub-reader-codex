# Bookshelf Home Balanced Library Design

## Goal

优化书架首页，使它在 300 本以上的个人书库中仍然优先服务“打开应用后快速继续阅读”，同时提供随时可用的搜索、轻量筛选和自动排序，并保持现有单层文件夹、手动书架顺序、触摸拖动与 Apple Books 式个人书房气质。

## Context

当前首页已经具备：

- “继续阅读”横向区域。
- 根书架中书籍与文件夹的统一手动顺序。
- 书籍和文件夹拖动排序。
- 拖动两本根层书籍创建文件夹。
- 根书架与文件夹之间移动书籍。
- 文件夹内排序、重命名和空文件夹清理。
- 固定封面尺寸、两行名称和轻量木质书架背景。

现有结构适合较小书库，但当书库达到 300 本以上时，仅靠手动网格和单层文件夹难以快速定位书籍。“继续阅读”与完整书架都使用纵向封面卡片，也会占用较多首屏高度。

## Confirmed product constraints

- 手机竖屏是唯一主要使用场景。
- 视觉基准使用 iPhone 14 Pro Max 的 430 × 932 CSS px 逻辑视口。
- 首页优先级依次为：继续阅读、搜索定位、浏览与整理。
- 搜索、最近记录和单层文件夹共同承担找书，不引入多层分类树。
- 文件夹只能存在一层，只能包含书籍，禁止文件夹嵌套。
- 根书架的手动顺序继续作为唯一可编辑的权威顺序。
- 搜索、筛选和自动排序都是只读派生视图，不得隐式修改手动顺序或文件夹归属。
- 宽屏设备继续限制内容最大宽度并居中，不扩展为桌面管理后台。

## Scope

- 将首页调整为“紧凑继续阅读 + 常驻搜索 + 首屏露出完整书架”的平衡布局。
- 增加覆盖全部书籍和单层文件夹的只读目录数据。
- 支持按书名、作者和文件夹名搜索。
- 支持“全部 / 最近添加 / 文件夹”快捷视图。
- 支持手动顺序、最近阅读、最近添加、书名和作者排序。
- 明确可拖动与只读状态的切换规则。
- 为目录加载、搜索无结果和不同空状态提供独立反馈。
- 为 300–500 本书库建立性能、可访问性和浏览器验收标准。

## Non-goals

- 不增加文件夹嵌套。
- 不增加标签、智能书单、评分或多条件高级查询。
- 不重写现有 DnD 状态机。
- 不改变文件夹打开浮层的基本信息架构。
- 不实现服务端逐字搜索、分页或全文检索。
- 不把首页拆成“阅读首页”和独立“资料库”两个导航层级。
- 不为桌面宽屏设计后台式侧栏或表格界面。
- 第一版不虚拟化可拖动书架，避免破坏跨行拖动和碰撞测量。

## Chosen approach

采用“阅读与书架平衡”方案。

相比只做搜索入口的最小改良，它能在首屏同时建立继续阅读、检索与完整书架的位置；相比拆分独立资料库，它保留轻量、直接的个人阅读动线，不增加导航层级。

## Information hierarchy

### Header

- 保留小号 Library eyebrow 和“我的书架”主标题。
- 右侧保留上传按钮。
- 不再增加独立圆形搜索按钮，避免与下方常驻搜索框重复。

### Sticky search

- 搜索框位于标题下方，高度不小于 44px。
- 初始提示显示可搜索书籍数量，例如“搜索 326 本书”。
- 页面向下滚动后，搜索框吸附在顶部安全区下方；标题正常滚出视口。
- 搜索范围包括所有根层书籍、文件夹内书籍和根层文件夹名称。

### Continue reading

- 使用横向宽卡片代替完整纵向封面卡片。
- 每张卡片包含较小封面、书名、阅读进度和最近阅读时间。
- 数据仍按阅读进度更新时间倒序，最多使用现有接口返回的 10 项。
- 横向区域完整显示第一张卡片并露出下一张，表达可横向滑动。
- 继续阅读不参与根书架排序，也不改变书籍文件夹归属。

### Library toolbar

- 标题显示“我的书架”和当前结果数量。
- 快捷视图为“全部 / 最近添加 / 文件夹”。
- 排序入口提供“手动顺序 / 最近阅读 / 最近添加 / 书名 / 作者”。
- 控件保持移动端轻量表现，不增加桌面式工具栏。

### Shelf grid

- 保持现有 110 × 165px 封面和约三列布局。
- 书名和文件夹名最多两行，超出省略。
- 文件夹与书籍保持相同卡片尺寸和根层视觉权重。
- 保留克制的浅木色背景、层板和封面阴影。
- 430 × 932 首屏应同时出现搜索、继续阅读和至少一整排书架。

## Interaction model

### Search

1. 聚焦搜索框后进入搜索状态并暂时隐藏继续阅读区域。
2. 输入内容后在浏览器本地即时匹配书名、作者和文件夹名，不发送逐字网络请求。
3. 非空搜索词始终搜索完整目录，不受进入搜索前的快捷视图限制。
4. 根层文件夹名称命中时显示文件夹卡片。
5. 文件夹内书籍命中时显示书籍卡片和“位于‘文件夹名’”上下文，点击书籍直接进入阅读器。
6. 搜索状态保留当前查询；打开书籍再返回时仍显示原结果。
7. 点击任一快捷视图会清空搜索并进入对应视图。
8. 取消或清空搜索后恢复进入搜索前的视图、排序和滚动位置。

### Views and sorting

- “全部 + 手动顺序”显示根书架中的书籍与文件夹，顺序来自 /api/folders/shelf。
- “最近添加”展示完整目录中的书籍，默认按 createdAt 倒序；文件夹内书籍显示所属文件夹。
- “文件夹”仅展示根层文件夹。
- 最近阅读、最近添加、书名和作者排序都只改变当前呈现顺序。
- 手动顺序只在“全部”视图有意义；离开“全部”后界面明确显示只读状态。
- 返回“全部 + 手动顺序”时恢复原根书架顺序，不把派生顺序写回服务器。

各状态的数据集和排序能力固定如下：

| 状态 | 数据集 | 默认顺序 | 可选排序 |
| --- | --- | --- | --- |
| 全部 | 根书架书籍与根层文件夹 | 手动顺序 | 手动、最近阅读、最近添加、书名、作者 |
| 最近添加 | 全部书籍，包含文件夹内书籍 | 最近添加 | 最近添加、最近阅读、书名、作者 |
| 文件夹 | 根层文件夹 | 根书架中的手动顺序 | 不显示排序入口 |
| 搜索 | 全部匹配书籍及名称命中的根层文件夹 | 书名或文件夹名 | 最近阅读、最近添加、书名、作者 |

“全部”视图使用自动排序时仍保留根层卡片结构，不展开文件夹。文件夹的自动排序键按以下规则计算：

- 书名排序使用文件夹名。
- 作者排序将文件夹放在有作者的书籍之后，文件夹之间按名称排序。
- 最近添加使用文件夹内最新书籍的 createdAt；没有书籍的文件夹不会存在。
- 最近阅读使用文件夹内最新的 readingUpdatedAt；从未阅读的文件夹排在已阅读项目之后。

### Drag eligibility

同时满足以下条件时才挂载可拖动书架：

| 条件 | 必须值 |
| --- | --- |
| 搜索词 | 空 |
| 快捷视图 | 全部 |
| 排序方式 | 手动顺序 |

任一条件不满足时：

- 使用只读卡片网格，不挂载 SortableContext 或拖动 listeners。
- 显示文字或图标提示“只读视图，不会改变手动书架顺序”。
- 不显示抓取光标，也不允许拖动删除、创建文件夹或移动归属。

### Folder behavior

- “文件夹”视图打开现有 FolderOverlay。
- 搜索到文件夹时使用相同打开行为。
- 文件夹内书籍仍允许现有排序和移出操作。
- 后端现有“文件夹只能包含书籍”的数据和接口校验继续作为最终约束。
- 本设计不增加任何创建子文件夹的入口或数据表示。

## Data architecture

### Existing authoritative sources

- GET /api/folders/shelf：根书架手动顺序和文件夹预览的唯一权威来源。
- GET /api/reading/recent：继续阅读区域的数据来源。
- 现有文件夹和书架写接口：继续负责所有排序与归属变更。

### New read-only catalog

增加 GET /api/books/catalog，并在 books router 中注册于 GET /api/books/:id 之前。

接口通过 books 左连接 folders 和 reading_progress，返回全部根层及文件夹内书籍：

    {
      "books": [
        {
          "id": 1,
          "folderId": 3,
          "folderName": "历史",
          "title": "万历十五年",
          "author": "黄仁宇",
          "coverUrl": "/covers/...",
          "createdAt": "2026-07-18T00:00:00.000Z",
          "readingProgress": 0.42,
          "readingUpdatedAt": "2026-07-18T01:00:00.000Z"
        }
      ]
    }

其他 formatBook 字段可保持兼容，但首页目录计算只依赖上述字段。

该接口：

- 只读，不修改数据库。
- 不改变现有 GET /api/books 的根层书籍语义。
- 不需要数据库迁移；folder_id、created_at、author 和 reading_progress 已存在。
- 对 300–500 行数据直接返回，不增加分页或服务端搜索。

### Client loading

useShelfData 分别维护 shelf、recentReading 和 catalog 的数据与加载状态：

- shelf 是首页的关键请求；失败时显示根书架错误和重试。
- recentReading 失败时隐藏继续阅读，不影响书架。
- catalog 失败时禁用搜索、快捷视图和自动排序，并在搜索区域提供重试；根书架和阅读入口仍可使用。
- 上传、删除、重命名或移动书籍成功后，在刷新 shelf 的同时刷新 catalog 和 recentReading，避免派生视图陈旧。

请求失败不应由一个共享 error 字符串覆盖其他已成功区域。

## Component boundaries

### LibraryHome

只负责组合 Header、LibrarySearchBar、ContinueReadingSection、LibraryViewToolbar 和 LibraryGrid，不直接实现搜索或排序算法。

### LibrarySearchBar

- 作为受控组件呈现输入、清空、取消和 sticky 表现，通过回调提交用户意图。
- 接收目录加载状态与结果数量。
- 通过语义 label 暴露“搜索书名、作者或文件夹”。

### ContinueReadingSection

- 保留现有数据与打开书籍回调。
- 改为紧凑宽卡片表现。
- 在 searchMode 时折叠。

### LibraryViewToolbar

- 呈现结果标题、数量、快捷视图、排序方式和只读提示。
- 使用 aria-pressed 表达快捷视图。
- 不直接修改 shelfItems。

### LibraryGrid

- editable 为 true 时使用现有 SortableShelfItem 和 SortableContext。
- editable 为 false 时使用复用 ShelfItemCover 的只读卡片。
- 搜索结果书籍可携带 folderName 上下文。
- 两种网格保持相同封面尺寸和点击打开行为。

### useLibraryView

接收 shelfItems 和 catalogBooks，集中持有首页的临时 query、view、sort、searchFocused 及进入搜索前的逻辑快照，返回：

- query、view、sort 及对应操作函数
- visibleItems
- resultCount
- searchMode
- editable
- modeLabel

搜索文本规范化、Intl.Collator 排序和派生列表使用 memoization。该 hook 不发送请求、不写服务器，也不复制书架权威数据；它只持有可丢弃的界面状态。LibraryHome 使用 scroll 容器 ref 在进入和退出搜索时保存、恢复滚动位置。

## Loading, error and empty states

- 首次根书架加载继续使用固定尺寸 skeleton，避免布局跳动。
- 目录加载期间搜索框显示加载提示并暂时禁用。
- 目录失败显示“搜索目录加载失败”和重试按钮，不替换正常书架。
- 空书架显示导入 EPUB 的明确操作。
- 无搜索结果显示当前查询、清空操作和“尝试书名、作者或文件夹名称”提示。
- 无文件夹时解释可通过拖动两本根层书籍创建文件夹。
- 无最近阅读时隐藏整个继续阅读区域。
- 上传和保存顺序使用不会推动页面布局的状态提示，并通过 aria-live 播报。

## Performance

- 目录首次加载后，输入搜索词不再请求服务器。
- 300–500 本数据的搜索、筛选和排序目标在代表性移动浏览器中 100ms 内完成。
- 非首屏封面使用 loading="lazy" 和 decoding="async"。
- 派生只读网格不挂载拖动 sensors 或 sortable items。
- 手动书架第一版保留完整 DOM，以保证现有跨行拖动和碰撞测量正确。
- 如果真实书库超过该范围后出现可验证瓶颈，再单独设计与 DnD 兼容的窗口化方案；本轮不提前引入。

## Accessibility and responsive behavior

- 搜索、筛选、排序、上传和取消操作的触摸区域不小于 44px。
- sticky 搜索框避开 env(safe-area-inset-top)。
- 页面底部内容避开 env(safe-area-inset-bottom) 和 Home Indicator。
- 视图、结果数量、加载和只读状态通过可访问状态区域播报。
- 可拖动与只读除颜色外同时使用文字或图标区分。
- 保留键盘拖动、focus-visible 和 prefers-reduced-motion 支持。
- 320px 窄屏按现有规则等比缩小封面；宽屏仍限制最大内容宽度为 760px。

## Error handling

- catalog 请求失败不清空最近一次成功目录，但标记数据可能陈旧并提供重试。
- shelf 请求失败不使用 catalog 结果伪装成可编辑书架。
- 自动排序字段缺失时使用稳定回退：空作者排在有作者之后，相同值按标题再按 id。
- 删除或移动后目录中已不存在的搜索结果立即移除。
- 打开搜索结果时若书籍已被外部文件同步删除，沿用现有“书籍不存在”处理并刷新 shelf、catalog 和 recentReading。
- 目录接口继续依赖现有数据库可用性检查，数据库不可用时返回 503。

## Testing

### Pure and component tests

- useLibraryView 覆盖书名、作者、文件夹名和文件夹内书籍搜索。
- 覆盖中文文本、空作者、相同标题和稳定排序。
- 覆盖 editable 条件矩阵，证明只有“空搜索 + 全部 + 手动顺序”为 true。
- 覆盖聚焦搜索隐藏继续阅读、取消恢复视图与滚动位置。
- 覆盖快捷视图清空搜索、搜索结果文件夹上下文、无结果和目录错误重试。
- 覆盖 aria-pressed、结果数量和模式状态播报。

### Server tests

- catalog 返回根层和文件夹内全部书籍。
- folderId 和 folderName 关联正确。
- 未阅读书籍的 readingProgress 和 readingUpdatedAt 为 null。
- 已阅读书籍正确返回进度和更新时间。
- catalog 不改变根书架顺序、文件夹归属或任何数据库时间字段。
- GET /api/books/:id 和现有 GET /api/books 行为保持不变。

### Browser and regression tests

- 使用 430 × 932、320px 窄屏和 760px 最大内容宽度验证布局。
- 300–500 条目录 fixture 验证本地结果更新时间和无逐字请求。
- 回归上传、打开阅读器、根书架排序、创建文件夹、拖入/移出、文件夹内排序和拖动删除。
- 验证自动排序或搜索状态不挂载可触发的拖动行为。
- 验证目录请求失败时仍能继续阅读和操作根书架。

### Real-device ownership

- iOS 和 Android PWA 的安全区、sticky 搜索、触摸滚动及长按拖动由用户在实现构建上完成最终实机验收。
- 实现方负责交付实机测试步骤和构建标识，但不得在用户确认前声称实机验收通过。
- 除实机体验外的自动化、模拟移动视口和桌面浏览器验证由实现方完成。

## Acceptance criteria

- 430 × 932 首屏同时出现搜索、继续阅读和至少一整排书架。
- 最近阅读书籍可一次点击继续。
- 300–500 本目录的本地搜索和排序在验收环境中 100ms 内更新，且不产生逐字网络请求。
- 文件夹内书籍可通过书名或作者搜索，并显示所属文件夹。
- 只有“全部 + 手动顺序 + 空搜索词”可拖动。
- 退出任何派生视图后，根书架顺序与书籍归属保持不变。
- 单层文件夹约束在界面、数据表示和后端接口中持续成立。
- catalog 失败不阻止继续阅读、打开根层书籍或整理手动书架。
- 所有非实机测试通过后，功能可交付用户进行 iOS/Android PWA 最终验收。
